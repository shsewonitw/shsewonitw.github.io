# Docker 컨테이너와 이미지의 차이점

## Docker란?
Docker는 소프트웨어를 컨테이너라는 격리된 환경에서 실행할 수 있게 해주는 플랫폼입니다. 컨테이너를 사용하면 애플리케이션과 그 의존성을 함께 패키징할 수 있어, 다양한 플랫폼에서 일관된 환경을 제공합니다.

## Docker 이미지
### 정의
Docker 이미지는 컨테이너를 만들기 위한 템플릿입니다. 해당 이미지에는 응용 프로그램, 라이브러리, 그리고 실행에 필요한 모든 파일이 포함되어 있습니다. 이미지는 읽기 전용이며 변경이 불가능한 상태입니다.

### 주요 특징
- **정적 콘텐츠**: 이미지는 실행이 가능하도록 모든 파일과 필수 설정을 포함하고 있습니다.
- **버전 관리**: 이미지는 레이어(layer)로 구성되어 있어, 수정사항을 쉽게 관리할 수 있습니다. 새로운 레이어를 추가하여 버전을 관리합니다.
- **Dockerfile**: 이미지는 코드로 구성할 수 있습니다. Dockerfile을 작성하여 이미지 빌드를 자동화할 수 있습니다.

### 예시
아래는 간단한 Dockerfile 예시입니다. 이 파일은 Python 애플리케이션을 위한 이미지를 생성합니다.

```dockerfile
# 기본 이미지
FROM python:3.9-slim

# 작업 디렉터리 생성
WORKDIR /app

# 파일 복사
COPY requirements.txt .

# 의존성 설치
RUN pip install -r requirements.txt

# 애플리케이션 복사
COPY . .

# 실행 명령어
CMD ["python", "app.py"]
```

## Docker 컨테이너
### 정의
컨테이너는 이미지에서 실행되는 인스턴스입니다. 즉, 이미지를 기반으로 만들어진 실제 실행 중인 환경이라고 볼 수 있습니다.

### 주요 특징
- **동적 상태**: 컨테이너는 실행 중에 상태를 변경할 수 있으며, 데이터와 설정을 추가하거나 수정할 수 있습니다. 
- **경량화**: 컨테이너는 호스트 시스템의 커널을 공유하므로, 더 적은 리소스를 사용하고 더 빠르게 시작할 수 있습니다.
- **독립적**: 컨테이너는 서로 독립적으로 실행되며, 격리된 환경에서 작업하는 것을 보장합니다.

### 예시
Docker 이미지를 기반으로 컨테이너를 실행하는 명령어는 다음과 같습니다.

```bash
docker run -d --name my-python-app my-python-image
```

위의 명령에서 `-d`는 백그라운드에서 실행하라는 의미이며, `--name` 옵션을 통해 컨테이너의 이름을 지정할 수 있습니다.

## 이미지와 컨테이너의 주요 차이점

| 항목        | Docker 이미지                     | Docker 컨테이너                     |
|-------------|----------------------------------|------------------------------------|
| 정의        | 실행 가능한 템플릿               | 실행 중인 인스턴스                   |
| 상태        | 읽기 전용                          | 동적 상태 (읽기 및 쓰기 가능)      |
| 생성 방법   | Dockerfile을 통해 생성            | `docker run` 명령어로 생성        |
| 유효성      | 변경 불가                        | 변형 가능                           |
| 사용 예     | 애플리케이션 배포 및 환경 정의   | 애플리케이션 실행                   |

## 요약
Docker 이미지는 애플리케이션을 실행하기 위한 템플릿이며, Docker 컨테이너는 그러한 템플릿을 기반으로 실행된 실제 인스턴스입니다. 이미지는 변하지 않는 상태를 유지하는 반면, 컨테이너는 실행 중에 변경될 수 있습니다. 두 개념은 Docker의 핵심 요소로, 컨테이너화를 통해 개발 환경을 보다 통합적이고 효율적으로 만들어 줍니다.
